## Structuring Spaces

You organize documentation by creating dedicated spaces for projects. Each space contains folders and documents. This keeps related content together and searchable.

## Using Folders and Tags

Create folders within spaces to group docs. Apply tags like `api`, `user-guide` for cross-referencing.

<Steps>
  <Step title="Create Folder" icon="folder">

    Right-click in space and select `New Folder`.

    ```javascript
    // API to create folder
    const folder = await client.folders.create({
      spaceId: 'space123',
      name: 'API Docs',
      parentId: null
    });
    ```

  </Step>
  <Step title="Add Tags" icon="tag">

    Edit document properties and assign tags.

    ```yaml
    # Example tag structure
    document:
      id: doc456
      tags:
        - api
        - v1.0
    ```

  </Step>
  <Step title="Search by Tag" icon="search">

    Use the search bar with `#tag` syntax.

  </Step>
</Steps>

<Tabs>

## Best Practices

Limit folder depth to three levels. Use tags for metadata like status (`draft`, `published`). Regularly audit organization to maintain clarity.

<CodeGroup tabs="CLI,API">
  ```bash
  # CLI to tag document
  paperguide doc tag doc456 --tags "api,v1.0"
  ```
  ```javascript
  // API tagging
  await client.documents.update(docId, { tags: ['api', 'v1.0'] });
  ```
</CodeGroup>

<ExpandableGroup>
  <Expandable title="How many spaces should I create?">

    One per major project or team to avoid overload.

  </Expandable>
  <Expandable title="Can tags be hierarchical?">

    Use prefixes like `api:auth` for pseudo-hierarchy.

  </Expandable>
</ExpandableGroup>
