## Setting Permissions

Assign roles like viewer, editor, admin per space or document. Use the settings panel to manage.

<Steps>
  <Step title="Invite User" icon="mail">

    Enter email in the invite section.

    ```javascript
    // API invite
    await client.spaces.invite(spaceId, {
      email: 'collab@example.com',
      role: 'editor'
    });
    ```

  </Step>
  <Step title="Set Role" icon="settings">

    Choose from dropdown: viewer, editor, admin.

    ```python
    # Python role update
    client.spaces.update_role(space_id, user_id, 'admin')
    ```

  </Step>
  <Step title="Review Access" icon="eye">

    Audit members in space settings.

  </Step>
</Steps>

<Tabs>
  <Tab title="Version History" icon="clock">

    View and revert changes.

    ````javascript
    // Fetch history
    const history = await client.documents.history(docId);
    history.forEach(version => {
      console.log(version.timestamp, version.changes);
    });
    `````

  </Tab>

## Maintenance Workflows

Schedule reviews and use notifications for updates. Export docs for backups.

<CodeGroup tabs="Export,Backup">
  ```bash
  # CLI export
  paperguide space export my-space --format md
  ```
  ```javascript
  // API backup
  const backup = await client.spaces.export(spaceId, 'markdown');
  ```
</CodeGroup>
